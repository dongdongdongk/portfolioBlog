# TalentSwap

<div className="project-overview">

## 재능 교환 플랫폼 & 커뮤니티 서비스

TalentSwap은 단독개발한 재능 교환 웹 플랫폼으로 주요 기능은 Supabase Auth 기반 인증 시스템, 1:1 재능교환 매칭, 그룹 학습 세션, 유료 레슨 마켓플레이스, 실시간 채팅, 커뮤니티 게시판입니다. 가상 스크롤링으로 성능 최적화, Row Level Security로 데이터 보안 강화, Realtime 기반 실시간 통신을 구현했습니다.

**프로젝트 링크**: [https://talentswap.roono.net/](https://talentswap.roono.net/)

</div>

<div className="tech-stack">

### 사용 기술

- **Frontend**: Next.js 14(App Router), TypeScript, SCSS + CSS Modules, Zustand, Axios, Recharts, React Icons
- **Backend**: Node.js, Next.js API Routes, PostgreSQL(Supabase), Prisma, NextAuth.js, bcryptjs
- **DevOps & 배포**: Vercel, Cloudinary, Vercel Analytics, Nodemailer

## 핵심 문제상황

<div className="problem-solution">

<div className="challenge-item">

### 대용량 데이터 렌더링 최적화 및 메모리 사용량 제어

- **문제**: 게시글 로딩 시 메모리 사용량이 500MB를 초과하며, 스크롤 성능이 현저히 저하되어 사용자 경험이 악화됨.
- **기술 해결**: react-window 라이브러리를 활용한 가상 스크롤링으로 DOM 노드 수를 최대 50개로 제한하고, Intersection Observer API로 기존 스크롤 이벤트 대신 더 효율적인 감지 방식을 적용. React.memo와 useMemo를 통한 컴포넌트 리렌더링 최적화 및 복잡한 계산 결과 캐싱, 디바운싱 기법으로 빠른 스크롤 시 API 호출 빈도를 300ms로 제한하여 성능을 최적화
- **결과**: 메모리 사용량 70% 감소 (500MB → 150MB), 스크롤 성능 개선으로 60fps 유지, 모바일 안정성 확보

</div>

<div className="challenge-item">

### 실시간 채팅 시스템 안정성 확보 및 메시지 순서 보장

- **문제**: WebSocket 연결 불안정으로 메시지 누락 발생, 브라우저 탭 포커스 변경 시 실시간 연결 끊김, 사용자 간 메시지 전송 순서가 보장되지 않아 대화 맥락 혼란.
- **기술 해결**: Supabase Realtime의 PostgreSQL 실시간 변경 감지 구독 시스템으로 WebSocket 대신 안정적인 메시지 동기화를 구현하고, 브라우저 포커스 이벤트 감지를 통한 자동 재연결 시스템을 구축했습니다. 클라이언트 측 메시지 큐 시스템으로 전송 실패 시 임시 저장 후 재시도하며, 서버 타임스탬프 기반 정렬을 통해 메시지 순서를 보장했습니다.
- **결과**: 메시지 전송 성공률 99.8% 달성, 평균 지연시간 50ms, 연결 유지율 99.2%

</div>
