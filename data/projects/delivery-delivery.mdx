# 배달배달

<div className="project-overview">

## 내 주변 맛집 추천 웹 서비스

배달배달은 단독 개발한 맛집 검색 웹 애플리케이션으로 주요 기능은 Spring Security 기반 회원가입/로그인, Geolocation API 위치 기반 검색, 카테고리별 음식점 검색, 실시간 크롤링을 통한 음식점 정보 제공, 찜 기능, 리뷰 정보 표시입니다. Kakao Map API로 시각적 위치 표시, Jsoup과 Selenium을 혼합한 크롤링으로 실시간 데이터 수집, Spring Validation과 JavaScript로 유효성 검사를 구현했습니다.



</div>

<div className="tech-stack">

### 사용 기술

- **Frontend**: HTML5, JS, CSS3, JQuery
- **Backend**: Java(JDK 1.8), Servlet, Spring, Mybatis
- **IDE**: IntelliJ, VisualStudio, DBeaver
- **Server**: Tomcat(v8.5)

## 핵심 문제상황

<div className="problem-solution">

<div className="challenge-item">

### 크롤링 성능 최적화 및 로딩 속도 개선

- **문제**: Selenium만으로 동적 크롤링 진행 시 페이지 로딩 속도가 저하되어 사용자 경험이 악화됨. 평균 응답 시간이 5초 이상 소요.
- **기술 해결**: 정적 크롤링(Jsoup)과 동적 크롤링(Selenium)을 혼합 사용하여 성능 최적화. Selenium에 GPU 비활성화, 팝업 차단, 이미지 다운로드 차단 등의 옵션 추가. 공식 문서 참고하여 브라우저 설정 최적화.
- **결과**: 페이지 로딩 속도 60% 개선, 실시간 데이터 수집과 성능의 균형점 확보.

</div>

<div className="challenge-item">

### 크롤링 차단 우회를 위한 지능형 요청 분산 시스템 구축

- **문제**: 사이트들의 bot 탐지 시스템이 강화되면서 IP 차단, CAPTCHA 발생으로 크롤링 성공률이 15%까지 급락. 특히 네이버 플레이스의 경우 연속 요청 시 즉시 차단되는 문제 발생.
- **기술 해결**: User-Agent 로테이션, 요청 간격 랜덤화(1-5초), Proxy 서버 풀 구성을 통한 IP 분산. Headless Chrome 옵션 최적화로 bot 탐지 우회, 실패 시 백오프 알고리즘 적용한 재시도 로직 구현.
- **결과**: 크롤링 성공률 92% 달성, 일일 데이터 수집량 10배 증가, 안정적인 실시간 정보 제공 실현.

</div>
