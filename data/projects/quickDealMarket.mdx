# QuickDealMarket (중고거래 플랫폼)

<div className="project-overview">

## E-Commerce Platform

### 📋 프로젝트 정보

QuickDealMarket은 단독 개발한 전자상거래 웹 애플리케이션으로 주요 기능은 JWT 이메일 인증 회원가입/로그인, ReduxToolkit 기반 장바구니, Kakao Pay API 결제, 쿠폰 할인, 이벤트 상품 카운트다운, WebSocket 1:1 채팅입니다. Multer로 파일 업로드, ErrorHandler와 미들웨어로 에러 처리, ProtectedRoute로 보안을 구현했으며, React-Toastify와 Lottie로 사용자 경험을 개선했습니다.

</div>

<div className="tech-stack">

### 🛠️ 사용 기술

- **Frontend**: React, Tailwind CSS, ReduxToolkit
- **Backend**: Node Express, Mongoose
- **Database**: MongoDB
- **Deployment**: AWS (EC2) * 현재는 무료 서비스 Koyeb & Firebase 로 전환 (서비스 종료)

## 핵심 문제상황

<div className="problem-solution">

<div className="challenge-item">

### 1. React.lazy와 Suspense 적용 전 초기 로딩 시간 지연

- **문제**: 초기 페이지 로드 시간이 평균 2.5초로 사용자 경험이 저하됨. Performance 탭 분석 결과, 대량 컴포넌트 로딩이 원인.
- **기술 해결**: React.lazy로 무거운 컴포넌트를 동적 로드하고, Suspense로 로딩 상태 관리 및 대체 UI 표시. Webpack 번들 분석으로 불필요한 의존성 제거.
- **결과**: 초기 로딩 시간 단축(2.5초→0.8초).

</div>

<div className="challenge-item">

### 2. 외부 API 호출 시 CORS 오류

- **문제**: Kakao Pay API 호출 시 동일 출처 정책(Same-Origin Policy)으로 요청 차단, CORS 오류 발생.
- **기술 해결**: MDN 문서 참고 후 Express 서버에 cors 라이브러리 적용, 허용 도메인 설정, 로컬 테스트용 프록시 추가, 배포 환경에서 서버 헤더 조정.
- **결과**: CORS 오류 해결, 안정적인 외부 API 통신 구현.

</div>
